# 09 两电机四舵机机器人整合指南

> 本指南说明如何把前面的单模块例程整合成最终机器人工程：两个 MG4010-i10 RS485 电机和四个 PWM 舵机由同一块 ESP32-S3 控制。

## 1. 能否同时驱动

可以。ESP32-S3 的资源足够完成本课程最终工程：

| 设备 | 控制方式 | ESP32-S3 资源 |
|------|----------|---------------|
| MG4010-i10 电机 1 | RS485 厂家私有协议 | UART2 + RS485/供电转接板或 RS485 模块 |
| MG4010-i10 电机 2 | RS485 厂家私有协议 | 与电机 1 共用同一条 RS485 总线 |
| PWM 舵机 1-4 | 50Hz PWM | 4 路 LEDC PWM 输出 |
| 串口调试 | USB Serial | USB CDC 或板载 USB 转串口 |
| 网页控制 | Wi-Fi AP + WebServer | ESP32-S3 Wi-Fi |

两个 RS485 电机不是占用两套串口，而是并联在同一条 RS485 A/B 总线上。程序向不同电机 ID 发送命令，电机根据 ID 判断是否响应。

## 2. 推荐引脚分配

| 功能 | 默认 GPIO | 说明 |
|------|-----------|------|
| RS485 TX | GPIO17 | 接课程转接板 TTL-485 调试口 T；使用通用 RS485 模块时接 DI |
| RS485 RX | GPIO18 | 接课程转接板 TTL-485 调试口 R；使用通用 RS485 模块时接 RO |
| RS485 DE/RE（仅通用模块） | GPIO16 | 课程转接板不接；散装 RS485 模块才用它控制 DE/RE |
| 舵机 1 | GPIO10 | LEDC PWM 输出 |
| 舵机 2 | GPIO11 | LEDC PWM 输出 |
| 舵机 3 | GPIO12 | LEDC PWM 输出 |
| 舵机 4 | GPIO13 | LEDC PWM 输出 |

这些引脚已经按 ESP32-S3 开发板原理图核对，均从排针引出，可以作为本课程默认接线。接线时仍要看清 PCB 丝印，避免把信号线误接到 3.3V、5V 或 GND。

如果后续更换其他 ESP32-S3 开发板，可以修改示例代码顶部的 `RS485_TX`、`RS485_RX`、`RS485_DE_RE` 和 `SERVO_PINS`。课程转接板默认 `RS485_DE_RE = -1`，表示不使用方向控制脚；只有使用通用 RS485 模块时才改成具体 GPIO。修改时避开启动相关引脚、板载 Flash/PSRAM 占用引脚、UART0 下载调试引脚以及已经连接板载外设的引脚。

## 3. 电机 ID 要求

最终工程中两个 MG4010-i10 电机必须使用不同 ID，例如：

```text
左电机：ID 1
右电机：ID 2
```

如果两个电机 ID 相同，它们会同时响应同一条命令，读取状态时还可能同时回包，导致 RS485 总线冲突。电机 ID 应按厂家上位机或厂家文档提供的方法提前设置，设置完成后再接入机器人整机。

## 4. 供电和共地

最终工程至少包含三类电源关系：

```text
ESP32-S3：USB 或稳定 5V 输入
MG4010-i10 电机：独立 12V~24V 电源
PWM 舵机：独立 5V 电源
```

必须连接公共地：

```text
ESP32 GND
RS485/供电转接板或 RS485 模块 GND
电机电源 V-
舵机电源 GND
```

如果使用课程配套 RS485/供电转接板，公共地应以转接板 GND、电机电源负极、ESP32 GND 和舵机电源 GND 为检查重点。不要用 ESP32 的 3.3V 引脚给舵机或电机供电。多个舵机同时动作时电流会明显增大，舵机电源容量不足会造成抖动、复位或通信异常。

## 5. 例程能否直接移植

前面的例程适合分模块学习，但不能简单复制粘贴成最终工程。移植时按下面原则处理：

| 来源例程 | 可复用内容 | 整合时要调整 |
|----------|------------|--------------|
| 03 舵机基础控制 | 角度到 PWM 占空比、非阻塞更新 | 扩展为 4 个舵机数组 |
| 05 舵机网页控制 | 网页请求处理、舵机目标角度更新 | 不要在网页请求里直接做长时间动作 |
| 06 电机基础控制 | RS485 帧格式、校验、收发方向 | 函数增加电机 ID 参数 |
| 07 电机速度模式 | 速度单位和减速比换算 | 左右电机分别发送速度 |
| 08 电机位置模式 | 位置单位和速度限制 | 根据机构需求选择是否使用 |
| 09 电机网页控制 | 网页控制结构和状态刷新 | 状态读取要轮询，避免长阻塞 |

最终工程建议以 `示例代码/10_机器人整合控制/` 为主程序模板，再逐步加入网页、传感器和任务逻辑。

## 6. 主循环结构

最终工程的 `loop()` 应保持快速循环：

```text
读取串口或网页命令
更新四个舵机的目标角度
按需要向两个电机发送速度或位置命令
周期性读取电机状态
处理异常和安全停止
```

不要在 `loop()` 中使用长时间 `delay()`。舵机平滑动作、电机状态查询、网页刷新都应使用 `millis()` 做周期调度。

## 7. 同时动作的风险排查

### 7.1 四个舵机同时动不到位

这类问题通常不是 ESP32-S3 PWM 资源不够，而是供电或机械负载问题。

可能原因：

1. 舵机 5V 电源电流不足，四个舵机同时启动时电压下跌。
2. 舵机电源 GND 没有和 ESP32 GND 可靠共地。
3. 机械结构卡住，舵机已经到达力矩上限。
4. 舵机角度范围设置过大，实际机构无法到达目标角度。
5. 四个舵机同一时刻大幅改变目标角度，瞬时电流过大。

处理建议：

- 用万用表测舵机电源，四个舵机同时动作时 5V 不应明显下跌。
- 舵机电源旁并联 470μF 到 1000μF 电解电容，靠近舵机供电端。
- 先拆掉机械负载，只测试舵机本体是否能到位。
- 程序中限制每次只小步改变角度，不要从 0° 瞬间跳到 180°。
- `10_机器人整合控制` 已采用四舵机轮询更新，减少同时大幅动作造成的电流尖峰。

### 7.2 舵机动作时电机串口命令不响应

PWM 舵机和 RS485 串口在 ESP32-S3 上使用不同硬件外设，正常情况下不会因为舵机 PWM 输出而占用 UART。出现“舵机一动，电机不响应”时，优先排查电源和共地。

可能原因：

1. 舵机动作导致 5V 电源跌落，ESP32 复位或 RS485 转接板/模块工作异常。
2. 舵机电流经过了细长杜邦线或面包板，造成地线电压波动。
3. 电机电源负极、舵机电源负极、ESP32 GND、RS485 转接板/模块 GND 没有形成可靠公共地。
4. 程序中等待电机回包时间过长，导致主循环响应变慢。
5. 两个电机 ID 相同，回包冲突，表现为偶发无回复。

处理建议：

- 电机和舵机分开供电，但所有 GND 必须连接在一起。
- 舵机电源线尽量短，电流不要经过 ESP32 板载 5V 或面包板细线路径。
- RS485 A/B 使用双绞线更稳定，线较长时在总线末端加 120Ω 终端电阻。
- 先只接电机测试 RS485，再只接四个舵机测试 PWM，最后合并测试。
- `10_机器人整合控制` 在等待电机回包时仍会更新舵机，避免串口等待完全卡住舵机调度。

### 7.3 判断是供电问题还是程序问题

按下面顺序定位：

| 测试 | 现象 | 判断 |
|------|------|------|
| 只接四个舵机 | 舵机不到位或抖动 | 舵机供电、机械负载或角度范围问题 |
| 只接两个电机 | 电机无回复 | RS485 接线、ID、波特率或共地问题 |
| 舵机和电机都接上 | 单独都正常，合并后异常 | 多半是供电容量、地线布局或电磁干扰 |
| 合并后串口打印重启信息 | ESP32 掉电复位 | 供电跌落或地线干扰 |
| 第二个电机经常无回复 | 只有 ID 2 异常 | 电机 ID、A/B 接线分支或总线冲突 |

工程调试时不要一开始就让四个舵机和两个电机同时满负载动作。先低速、小角度、空载验证，再逐步增加动作幅度和机械负载。

## 8. 推荐调试顺序

1. 只接 ESP32，上传 LED 和串口例程，确认开发环境正常。
2. 只接一个舵机，测试 0°、90°、180°。
3. 接四个舵机，分别测试角度，确认供电不掉压。
4. 只接一个 RS485 电机，测试运行、停止、低速速度模式。
5. 接第二个 RS485 电机，确认两个电机 ID 不同。
6. 上传 `10_机器人整合控制`，先空载测试左右电机和四个舵机。速度命令会自动发送电机运行命令，仍建议第一次先低速测试。
7. 安装到机器人结构后，低速、小角度测试，再逐步增加动作范围。

## 9. 安全边界

- 上电前检查电机电源正负极和 RS485 A/B。
- 第一次测试不要让轮子或机构接触桌面边缘。
- 先用低速，例如输出轴 `30 dps` 或 `60 dps`。
- 舵机先测试 60° 到 120° 的小范围，再扩大到实际机构允许范围。
- 发现异常声音、发热、卡死、复位或通信失控时立即断电。
