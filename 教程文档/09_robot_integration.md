# 09 两电机四舵机机器人整合指南

> 本指南说明如何把前面的单模块例程整合成最终机器人工程：两个 MG4010-i10 RS485 电机和四个 PWM 舵机由同一块 ESP32-S3 控制。

## 1. 能否同时驱动

可以。ESP32-S3 的资源足够完成本课程最终工程：

| 设备 | 控制方式 | ESP32-S3 资源 |
|------|----------|---------------|
| MG4010-i10 电机 1 | RS485 厂家私有协议 | UART2 + RS485 模块 |
| MG4010-i10 电机 2 | RS485 厂家私有协议 | 与电机 1 共用同一条 RS485 总线 |
| PWM 舵机 1-4 | 50Hz PWM | 4 路 LEDC PWM 输出 |
| 串口调试 | USB Serial | USB CDC 或板载 USB 转串口 |
| 网页控制 | Wi-Fi AP + WebServer | ESP32-S3 Wi-Fi |

两个 RS485 电机不是占用两套串口，而是并联在同一条 RS485 A/B 总线上。程序向不同电机 ID 发送命令，电机根据 ID 判断是否响应。

## 2. 推荐引脚分配

| 功能 | 默认 GPIO | 说明 |
|------|-----------|------|
| RS485 TX | GPIO17 | ESP32 发送到 RS485 模块 DI |
| RS485 RX | GPIO18 | RS485 模块 RO 发送到 ESP32 |
| RS485 DE/RE | GPIO16 | 控制 RS485 收发方向，DE 和 RE 接在一起 |
| 舵机 1 | GPIO10 | LEDC PWM 输出 |
| 舵机 2 | GPIO11 | LEDC PWM 输出 |
| 舵机 3 | GPIO12 | LEDC PWM 输出 |
| 舵机 4 | GPIO13 | LEDC PWM 输出 |

如果实际开发板没有引出这些引脚，可以修改示例代码顶部的 `RS485_TX`、`RS485_RX`、`RS485_DE_RE` 和 `SERVO_PINS`。修改时避开启动相关引脚、板载 Flash/PSRAM 占用引脚以及已经连接板载外设的引脚。

## 3. 电机 ID 要求

最终工程中两个 MG4010-i10 电机必须使用不同 ID，例如：

```text
左电机：ID 1
右电机：ID 2
```

如果两个电机 ID 相同，它们会同时响应同一条命令，读取状态时还可能同时回包，导致 RS485 总线冲突。电机 ID 应按厂家上位机或厂家文档提供的方法提前设置，设置完成后再接入机器人整机。

## 4. 供电和共地

最终工程至少包含三类电源关系：

```text
ESP32-S3：USB 或稳定 5V 输入
MG4010-i10 电机：独立 12V~24V 电源
PWM 舵机：独立 5V 电源
```

必须连接公共地：

```text
ESP32 GND
RS485 模块 GND
电机电源 V-
舵机电源 GND
```

不要用 ESP32 的 3.3V 引脚给舵机或电机供电。多个舵机同时动作时电流会明显增大，舵机电源容量不足会造成抖动、复位或通信异常。

## 5. 例程能否直接移植

前面的例程适合分模块学习，但不能简单复制粘贴成最终工程。移植时按下面原则处理：

| 来源例程 | 可复用内容 | 整合时要调整 |
|----------|------------|--------------|
| 03 舵机基础控制 | 角度到 PWM 占空比、非阻塞更新 | 扩展为 4 个舵机数组 |
| 05 舵机网页控制 | 网页请求处理、舵机目标角度更新 | 不要在网页请求里直接做长时间动作 |
| 06 电机基础控制 | RS485 帧格式、校验、收发方向 | 函数增加电机 ID 参数 |
| 07 电机速度模式 | 速度单位和减速比换算 | 左右电机分别发送速度 |
| 08 电机位置模式 | 位置单位和速度限制 | 根据机构需求选择是否使用 |
| 09 电机网页控制 | 网页控制结构和状态刷新 | 状态读取要轮询，避免长阻塞 |

最终工程建议以 `示例代码/10_机器人整合控制/` 为主程序模板，再逐步加入网页、传感器和任务逻辑。

## 6. 主循环结构

最终工程的 `loop()` 应保持快速循环：

```text
读取串口或网页命令
更新四个舵机的目标角度
按需要向两个电机发送速度或位置命令
周期性读取电机状态
处理异常和安全停止
```

不要在 `loop()` 中使用长时间 `delay()`。舵机平滑动作、电机状态查询、网页刷新都应使用 `millis()` 做周期调度。

## 7. 推荐调试顺序

1. 只接 ESP32，上传 LED 和串口例程，确认开发环境正常。
2. 只接一个舵机，测试 0°、90°、180°。
3. 接四个舵机，分别测试角度，确认供电不掉压。
4. 只接一个 RS485 电机，测试运行、停止、低速速度模式。
5. 接第二个 RS485 电机，确认两个电机 ID 不同。
6. 上传 `10_机器人整合控制`，先空载测试左右电机和四个舵机。
7. 安装到机器人结构后，低速、小角度测试，再逐步增加动作范围。

## 8. 安全边界

- 上电前检查电机电源正负极和 RS485 A/B。
- 第一次测试不要让轮子或机构接触桌面边缘。
- 先用低速，例如输出轴 `30 dps` 或 `60 dps`。
- 舵机先测试 60° 到 120° 的小范围，再扩大到实际机构允许范围。
- 发现异常声音、发热、卡死、复位或通信失控时立即断电。
