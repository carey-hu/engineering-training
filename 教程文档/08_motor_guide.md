# 08 MG4010-i10 RS485 电机使用指南

> 本指南面向零基础学习者，目标是先安全接线，再理解厂家 RS485 协议，最后完成串口和网页控制。

---

## 1. 电机和协议版本

本课程使用的是 **MG4010-i10 RS485 版本电机**。厂家资料包含两类信息：

- [MG4010-i10电机手册.pdf](../硬件资料/技术文档/MG4010-i10电机手册.pdf)：电机供电、接口、减速比、上位机设置等。
- [MG4010-i10-R485协议.pdf](../硬件资料/技术文档/MG4010-i10-R485协议.pdf)：RS485 通信帧格式和控制命令。

需要特别注意：厂家 RS485 协议 **不是 Modbus-RTU 寄存器协议**。本课程示例代码使用厂家私有帧协议，帧头为 `0x3E`，通过命令字控制电机。

主要参数：

```text
供电电压：7.4V~32V，首次测试建议使用 24V
通信接口：RS485
默认波特率：115200
数据格式：8 数据位，无校验，1 停止位（8N1）
默认 ID：1
ID 范围：1~32
减速比：i10，即电机侧转 10 圈，输出轴约转 1 圈
编码器分辨率：18bit（手册标称）
```

## 2. 接线方法

### 2.1 电机接口

以厂家手册中的 RS485 接口定义为准：

| 电机接口 | 含义 |
|---------|------|
| V+ | 电源正极 |
| V- | 电源负极 |
| A/H | RS485-A |
| B/L | RS485-B |

部分配套线材可能用颜色区分。接线前以实际线材说明或万用表确认，不要只凭颜色判断。

### 2.2 推荐接线：使用 RS485/供电转接板

课程配套转接板原理图见：

[转接板原理图.pdf](../转接板原理图.pdf)

上机接线示意图见：

[电机转接板接线示意图.svg](../硬件资料/接线图/电机转接板接线示意图.svg)

建议学生优先使用这块转接板，而不是直接用散装 RS485 模块。转接板把 RS485 通信、电机电源输入、电机 A/B 总线和公共地集中到一块板上，能显著减少初学者接错电源或忘记共地的概率。

按转接板丝印核对下列连接：

| ESP32-S3 | 转接板 | 说明 |
|----------|--------|------|
| 5V | USB调试 5V | ESP32 可从 USB 调试口给转接板 5V 供电 |
| GND | USB调试 G | 与电机电源负极共地 |
| GPIO17 (TX) | TTL-485 调试 T | ESP32 UART 发送到转接板，TX 接 T |
| GPIO18 (RX) | TTL-485 调试 R | 转接板回传数据到 ESP32 UART，RX 接 R |

| 电机电源/电机 | 转接板 | 说明 |
|---------------|--------|------|
| 24V 电源正极 | 输入端子上孔 | 输入端子有防呆，上正下负 |
| 24V 电源负极 | 输入端子下孔 | 与 ESP32 GND 共地 |
| MG4010 B/L | 电机 6pin 的 B | 从板子正面看、接口在下方时，第 1 pin |
| MG4010 A/H | 电机 6pin 的 A | 从板子正面看、接口在下方时，第 2 pin |
| MG4010 V- | 电机 6pin 的 GND/GND | 从板子正面看、接口在下方时，第 3、4 pin |
| MG4010 V+ | 电机 6pin 的 24V/24V | 从板子正面看、接口在下方时，第 5、6 pin |

四个并排的电机 6pin 接口定义相同。从板子正面看、接口在下方时，每个 6pin 从左到右为：

```text
B    A    GND    GND    24V    24V
```

从原理图看，课程配套转接板的 RS485 芯片 `DI` 接 `TX3`，`RO` 接 `RX3`，`RE#` 和 `SHDN#` 已经通过 10k 上拉到 `5VIN`，收发方向由转接板处理。ESP32 不需要再接 GPIO16 控制 DE/RE。

上电前重点检查 `24VIN/VIN` 和 `GND` 不要接反，`A/B` 不要和电源端子混接。转接板上的 `5VIN`、`VCC` 等网络按实际板子丝印和课堂配套线使用，不要把它们当作电机 24V 输入。

### 2.3 替代接线：使用通用 RS485 模块

本教程示例默认使用如下引脚：

| ESP32-S3 | RS485 模块 | 说明 |
|----------|------------|------|
| GPIO17 | DI | ESP32 发送到 RS485 模块 |
| GPIO18 | RO | RS485 模块发送到 ESP32 |
| GPIO16 | DE 和 RE | 收发方向控制，两个引脚连在一起 |
| 3.3V | VCC | 优先使用 3.3V 兼容 RS485 模块 |
| GND | GND | 必须与电机电源负极共地 |
| RS485-A | A/H | 接电机 A/H |
| RS485-B | B/L | 接电机 B/L |

GPIO17、GPIO18、GPIO16 已在本教程使用的 ESP32 开发板排针上引出。使用课程配套转接板时只用 GPIO17/GPIO18；使用通用 RS485 模块时才额外使用 GPIO16 控制 DE/RE。这样避开了 UART0 调试/下载使用的 GPIO43/GPIO44，也避开了原生 USB 使用的 GPIO19/GPIO20 和启动配置相关引脚。

推荐使用支持 3.3V 逻辑的 SP3485、MAX3485 或明确标注 3.3V 可用的 RS485 模块。若使用 5V MAX485 模块，要确认 RO 输出不会把 5V 直接送入 ESP32 GPIO。

### 2.4 供电要求

```text
电机供电：独立 12V~24V 电源，首次测试建议 24V 2A 以上
ESP32 供电：USB 或稳定 5V
共地要求：ESP32 GND、RS485/供电转接板或 RS485 模块 GND、电机电源 V- 必须连接
```

不要用 ESP32 的 5V 或 3.3V 引脚给电机供电。电机启动电流较大，供电不足会导致电机抖动、ESP32 复位或通信异常。

## 3. 厂家 RS485 帧格式

单电机命令的基本格式：

```text
帧头     命令字   电机 ID   数据长度   帧命令校验   数据区   数据校验
0x3E     CMD      0x01      N          SUM          ...      SUM
```

校验规则：

- 帧命令校验：`帧头 + 命令字 + ID + 数据长度`，只保留低 8 位。
- 数据校验：数据区所有字节相加，只保留低 8 位。
- 多字节整数使用小端序，即低字节在前。

示例：让 ID 为 1 的电机进入运行状态。

```text
3E 88 01 00 C7
```

计算方式：

```text
0x3E + 0x88 + 0x01 + 0x00 = 0xC7
```

## 4. 常用命令

| 功能 | 命令字 | 数据长度 | 说明 |
|------|--------|----------|------|
| 电机关闭 | `0x80` | 0 | 关闭输出，电机仍可回复命令 |
| 电机停止 | `0x81` | 0 | 停止运动，不清除运行状态 |
| 电机运行 | `0x88` | 0 | 从关闭状态切换到运行状态 |
| 读取多圈角度 | `0x92` | 0 | 返回 `int64_t`，单位 0.01° |
| 清除错误 | `0x9B` | 0 | 错误原因解除后才能清除 |
| 读取状态 2 | `0x9C` | 0 | 返回温度、电流采样、速度、编码器 |
| 速度闭环 | `0xA2` | 4 | `int32_t speedControl`，单位 0.01dps |
| 多圈位置闭环 2 | `0xA4` | 12 | `int64_t angleControl` + `uint32_t maxSpeed` |
| 设置零点到 ROM | `0x19` | 0 | 写入 Flash，不要频繁使用 |

学习时建议先掌握 `0x88`、`0x81`、`0x9C`、`0xA2` 和 `0xA4`。

## 5. 减速比和单位换算

MG4010-i10 的减速比为 1:10。厂家协议里的速度和角度是 **电机侧** 数值，而你实际观察到的是 **输出轴**。

### 5.1 速度换算

协议速度单位为 `0.01dps/LSB`，dps 表示度每秒。

```text
电机侧速度 dps = 输出轴速度 dps × 10
协议原始值 = 电机侧速度 dps × 100
```

示例：希望输出轴以 36dps 转动。

```text
电机侧速度 = 36 × 10 = 360dps
协议原始值 = 360 × 100 = 36000
```

### 5.2 位置换算

协议位置单位为 `0.01°/LSB`，多圈位置命令使用电机侧角度。

```text
电机侧角度 = 输出轴角度 × 10
协议原始值 = 电机侧角度 × 100
```

示例：希望输出轴转到 90°。

```text
电机侧角度 = 90 × 10 = 900°
协议原始值 = 900 × 100 = 90000
```

## 7. 常见问题

### 7.1 电机没有任何回复

按顺序排查：

1. 电机是否有独立电源，电压是否在允许范围内。
2. ESP32 GND、RS485/供电转接板或 RS485 模块 GND、电机 V- 是否共地。
3. RS485 A/B 是否接反。
4. 电机 ID 是否为 1。
5. 电机波特率是否为 115200。
6. 使用转接板时，GPIO17(TX) 是否接 TTL-485 调试口 T，GPIO18(RX) 是否接 R，ESP32 5V/GND 是否接 USB调试 5V/G，24V 输入是否上正下负。
7. 使用通用 RS485 模块时，DE/RE 是否接到 GPIO16，模块是否支持 ESP32 的 3.3V 逻辑。

### 7.2 串口能发送，但电机不动

可能原因：

- 电机处于关闭状态，需要先发送 `0x88` 运行命令。
- 电机存在保护错误，需要先排除故障，再发送 `0x9B` 清除错误。
- 速度或位置目标过小，看起来像没有动作。
- 机械负载过大，电机进入保护或堵转。

### 7.3 电机方向和预期相反

厂家手册中方向与电机安装方向、上位机设置有关。初学阶段先接受“正负号控制两个方向”的概念，不要急着修改驱动内部方向参数。若项目需要统一方向，可在代码里对速度和角度乘以 `-1`。

### 7.4 设置零点要谨慎

`0x19` 会把当前位置作为零点写入驱动器 Flash。Flash 有擦写寿命，不要把它当作普通运行命令反复调用。练习时优先使用读取角度和相对目标，不建议频繁写 ROM 零点。

## 8. 学习建议

### 8.1 学习路线

| 阶段 | 内容 | 完成标志 |
|------|------|----------|
| 安全接线 | 电源、共地、A/B 接线、断电检查 | 上电前能独立检查接线 |
| 协议认识 | `0x3E` 帧、命令字、校验和 | 能说出一帧命令由哪些字段组成 |
| 基础控制 | 运行、停止、读取状态 | 能用串口让电机运行和停止 |
| 速度模式 | dps、减速比、正反转 | 能设置低速正转和反转 |
| 位置模式 | 角度单位、多圈位置、速度限制 | 能让输出轴转到指定小角度 |
| 网页控制 | AP 模式、HTTP 参数、状态刷新 | 能用浏览器控制电机 |

### 8.2 必须掌握的检查步骤

1. 上电前先检查电源正负极，确认没有短路。
2. 先打开串口监视器，再给电机供电。
3. 第一次发送命令先读状态，不要直接高速运行。
4. 速度从小到大增加，观察电机和电源是否异常。
5. 出现异常声音、发热、复位或失控时立即断电。

### 8.3 课程目标

完成本章后，你应能够：

1. 说清楚 RS485 和 UART 的区别。
2. 根据厂家协议组装一帧控制命令。
3. 理解校验和的作用。
4. 解释减速比对速度和角度的影响。
5. 使用串口和网页两种方式控制电机。
