## 环境搭建类问题

### Q1: 找不到 COM 口（Windows）

**现象：** Arduino IDE 的 Tools → Port 菜单是灰色或为空。

**原因：**
1. USB 线是充电线，不是数据线
2. 串口驱动未安装
3. 开发板未通电

**解决步骤：**

1. **检查 USB 线**
   - 换一根 USB 数据线（有数据传输功能）
   - 确认不是只能充电的线

2. **检查设备管理器**
   - 右键"此电脑" → 管理 → 设备管理器
   - 查看"端口（COM 和 LPT）"
   - 插拔开发板，观察是否有变化

3. **安装串口驱动**
   
   **如果看到未知设备或黄色感叹号：**
   
   - 查看芯片型号（开发板上的小芯片）
   - CH340/CH341：安装 [CH341SER](../硬件资料/驱动程序/CH341SER.EXE)
   - CH343：下载 [CH343SER](https://www.wch.cn/downloads/CH343SER_EXE.html)
   - 双击安装包，重启电脑

4. **检查开发板供电**
   - 开发板电源指示灯是否亮
   - 换一个 USB 口或电脑试试

**完整排查顺序：**
```
换USB线 → 看设备管理器 → 装驱动 → 重启 → 换USB口 → 换电脑
```

### Q2: 找不到串口（macOS）

**现象：** Tools → Port 菜单为空或灰色。

**解决步骤：**

1. **在终端检查串口**
   ```bash
   ls /dev/cu.*
   ```
   
   应该能看到：
   ```
   /dev/cu.usbserial-0001  ← ESP32串口
   ```

2. **如果看不到串口设备：**
   
   - 检查 USB 线（必须是数据线）
   - 安装 WCH 驱动：
     - CH340: https://www.wch.cn/downloads/CH34XSER_MAC_ZIP.html
     - CH343: https://www.wch.cn/downloads/CH343SER_MAC_ZIP.html
   
3. **安装驱动后重启 Mac**

4. **允许内核扩展**
   - 系统设置 → 隐私与安全性
   - 找到"江苏沁恒..."提示
   - 点击"允许"
   - 再次重启

**参考教程：** [02 macOS 安装教程](02_install_macos.md#3-安装-usb-串口驱动)

### Q3: 板卡包安装失败或下载很慢

**现象：** Boards Manager 一直显示"Downloading..."或报错。

**原因：**
1. 网络问题
2. 未使用国内镜像
3. Arduino IDE 版本过旧

**解决方法：**

1. **确认使用国内镜像**
   
   File → Preferences → Additional Boards Manager URLs
   
   应该包含：
   ```
   https://jihulab.com/esp-mirror/espressif/arduino-esp32/-/raw/gh-pages/package_esp32_index_cn.json
   ```

2. **选择带 `-cn` 后缀的版本**
   
   Boards Manager 中搜索 `esp32`，选择：
   ```
   esp32 by Espressif Systems (版本号-cn)
   ```

3. **清除缓存重试**
   
   **Windows:**
   ```
   删除目录：C:\Users\用户名\AppData\Local\Arduino15\staging
   ```
   
   **macOS:**
   ```bash
   rm -rf ~/Library/Arduino15/staging
   ```
   
   然后重新在 Boards Manager 中安装。

4. **更新 Arduino IDE**
   
   建议使用 Arduino IDE 2.3.0 或更高版本。

**快速下载链接：** [00 下载链接汇总](00_downloads_cn.md)

## 上传失败类问题

### Q4: 上传卡在 "Connecting........"

**现象：**
```
Connecting........_____....._____....._____.....
```
程序一直卡在连接，无法上传。

**解决方法（按优先级）：**

**方法 1：按住 BOOT 键上传（最常用）**

1. 点击 Arduino IDE 的"上传"按钮
2. 当看到底部输出 `Connecting` 时，**按住开发板上的 BOOT 键**
3. 看到 `Writing at...` 开始上传后，**松开 BOOT 键**
4. 等待上传完成

**方法 2：降低上传速度**

```
Tools → Upload Speed → 115200
```

然后重新上传。

**方法 3：检查 USB CDC 设置**

```
Tools → USB CDC On Boot → Enabled
```

确保已启用，然后重新上传。

**方法 4：检查串口占用**

- 关闭串口监视器（Serial Monitor）
- 关闭其他可能占用串口的程序（如串口助手、PuTTY）

**方法 5：重启开发板和 IDE**

1. 拔掉 USB 线，等待 5 秒
2. 插回 USB 线
3. 重启 Arduino IDE

---

### Q5: 上传成功但程序不运行

**现象：** 显示 "Done uploading"，但 LED 不闪烁或串口无输出。

**解决方法：**

1. **按 EN（或 RST）键复位**
   
   上传完成后，按一下开发板的 EN 或 RESET 键，让程序重新启动。

2. **检查 USB CDC On Boot 设置**
   
   ```
   Tools → USB CDC On Boot → Enabled
   ```
   
   如果之前是 Disabled，改为 Enabled 后需要重新上传。

3. **检查代码是否有 setup() 和 loop()**
   
   ```cpp
   void setup() {
     // 初始化代码
   }
   
   void loop() {
     // 主循环代码
   }
   ```

4. **查看串口输出**
   
   打开串口监视器（Ctrl+Shift+M），看是否有错误信息。

### Q6: 编译报错："GPIO2 was not declared"

**现象：**
```
error: 'GPIO2' was not declared in this scope
```

**原因：** 代码中使用了不正确的 GPIO 常量名。

**解决方法：**

**错误写法：**
```cpp
pinMode(GPIO2, OUTPUT);  // ❌ 错误
```

**正确写法：**
```cpp
pinMode(2, OUTPUT);      // ✅ 直接用数字
// 或
const int LED_PIN = 2;
pinMode(LED_PIN, OUTPUT); // ✅ 定义常量
```

**说明：** Arduino ESP32 中，GPIO 引脚直接用数字表示，不需要 `GPIO` 前缀。

## 硬件连接类问题

### Q7: 舵机抖动或不转

**现象：** 舵机到达目标位置后不停抖动，或发出"嗡嗡"声。

**最常见原因：供电不足**

**解决方法：**

1. **使用独立 5V 电源**
   
   ❌ 错误接线：
   ```
   舵机红线 → ESP32 的 5V 引脚
   ```
   
   ✅ 正确接线：
   ```
   舵机红线 → 外部 5V 电源正极（如 USB 充电器、稳压模块）
   舵机棕线 → 外部 5V 电源负极
   ESP32 GND → 外部 5V 电源负极（共地！）
   舵机黄线 → ESP32 GPIO13
   ```

2. **增加滤波电容**
   
   在舵机电源线附近并联 **470μF - 1000μF 电解电容**：
   ```
   电容正极 → 舵机红线（5V）
   电容负极 → 舵机棕线（GND）
   ```

3. **减少同时动作的舵机数量**
   
   如果有 4 个舵机，不要让它们同时从 0° 跳到 180°，改为错开启动：
   ```cpp
   setServo(1, 90);
   delay(200);
   setServo(2, 90);
   delay(200);
   setServo(3, 90);
   ```

4. **检查舵机是否卡住**
   
   拆掉机械负载，只测试舵机本体是否能平滑转动。

**完整排查流程：**
```
检查供电 → 万用表测电压 → 加电容 → 减少同时动作 → 检查机械负载
```

**参考教程：** [07 舵机指南 - 常见问题](07_servo_guide.md#5-常见问题与解决方法)

### Q8: 电机没有任何回复

**现象：** 串口输入命令后，显示"未收到有效回复"。

**按优先级排查：**

1. **检查电机 ID**
   
   - 代码中默认 ID 为 1
   - 用厂家上位机确认电机 ID
   - 如果 ID 不是 1，修改代码：
     ```cpp
     #define MOTOR_ID 0x02  // 改为实际 ID
     ```

2. **检查波特率**
   
   - 代码默认 115200
   - 用厂家上位机确认电机波特率
   - 修改代码：
     ```cpp
     #define RS485_BAUD 9600  // 改为实际波特率
     ```

3. **检查 RS485 A/B 接线**
   
   **使用课程转接板：**
   ```
   电机 A/H → 转接板 A
   电机 B/L → 转接板 B
   ```
   
   **使用通用 RS485 模块：**
   ```
   电机 A/H → RS485 模块 A
   电机 B/L → RS485 模块 B
   ```
   
   **注意：** A 和 B 接反会完全无响应，不是响应错误。

4. **检查共地**
   
   ```
   ESP32 GND → 转接板/模块 GND → 电机电源负极
   ```
   
   用万用表通断档测量是否真正连通。

5. **检查 TX/RX 接线**
   
   **使用课程转接板：**
   ```
   ESP32 GPIO17 (TX) → 转接板 TTL-485 调试口 T
   ESP32 GPIO18 (RX) → 转接板 TTL-485 调试口 R
   ```
   
   **使用通用 RS485 模块：**
   ```
   ESP32 GPIO17 (TX) → RS485 模块 DI
   ESP32 GPIO18 (RX) → RS485 模块 RO
   ESP32 GPIO16     → RS485 模块 DE 和 RE（连在一起）
   ```

6. **检查电源**
   
   - 电机需要 12V-24V 独立电源
   - 用万用表测量电机端子电压
   - 确认电源容量足够（至少 2A）

7. **检查 DE/RE 设置（仅通用模块）**
   
   使用通用 RS485 模块时，确认代码：
   ```cpp
   #define RS485_DE_RE 16  // 不是 -1
   ```
   
   使用课程转接板时：
   ```cpp
   #define RS485_DE_RE -1  // 转接板自动控制
   ```

**快速检查清单：**
```
☐ 电机 ID 正确
☐ 波特率正确（115200）
☐ A/B 接线正确（没有接反）
☐ 共地连接可靠
☐ TX/RX 接线正确
☐ 电机供电正常（12V-24V）
☐ DE/RE 设置正确（仅通用模块）
```

**参考教程：** [08 电机指南 - 常见问题](08_motor_guide.md#7-常见问题)

### Q9: ESP32 复位或自动重启

**现象：** 串口输出显示反复重启信息，或程序运行一会儿就复位。

**原因：**
1. 供电不足（最常见）
2. 短路或过载
3. 代码错误（如栈溢出）

**解决方法：**

1. **检查供电**
   
   - 不要从 USB 供电的同时给舵机/电机供电
   - 舵机和电机使用独立电源
   - USB 线质量差会导致压降

2. **检查共地**
   
   ```
   ESP32 GND → 舵机电源 GND → 电机电源 GND
   ```

3. **查看串口复位信息**
   
   打开串口监视器，记录复位信息：
   ```
   Brownout detector triggered  ← 电压跌落复位
   Core 1 panic'ed (...)        ← 代码错误
   ```
   
   根据错误类型排查。

4. **简化测试**
   
   - 只接 ESP32，不接舵机和电机
   - 上传简单代码（如 LED 闪烁）
   - 如果还复位，说明是供电或硬件问题

**参考教程：** [09 机器人整合 - 故障排查](09_robot_integration.md#7-同时动作的风险排查)

## 舵机控制类问题

### Q10: 舵机角度不准

**现象：** 设置 90° 但舵机没转到中间位置。

**原因：** 不同品牌舵机的脉宽范围略有差异。

**解决方法：**

在代码中调整脉宽范围：

```cpp
uint32_t angleToDuty(int angle) {
  angle = constrain(angle, 0, 180);
  
  // 原始范围
  // const int minUs = 500;
  // const int maxUs = 2500;
  
  // 调整后的范围（根据实际测试）
  const int minUs = 600;   // 如果 0° 转不到位，增大此值
  const int maxUs = 2400;  // 如果 180° 转不到位，减小此值
  
  int pulseUs = map(angle, 0, 180, minUs, maxUs);
  uint32_t maxDuty = (1UL << 16) - 1;
  return (uint32_t)((pulseUs * maxDuty) / 20000UL);
}
```

**调试步骤：**

1. 先测试 90°，观察是否在中间位置
2. 测试 0° 和 180°，观察是否到达机械极限
3. 根据偏差调整 minUs 和 maxUs

**参考教程：** [07 舵机指南 - 角度不准](07_servo_guide.md#53-舵机角度不准)

### Q11: 网页控制舵机时卡顿

**现象：** 点击网页按钮后，页面卡住几秒才恢复。

**原因：** ESP32 代码中使用了 `delay()` 阻塞。

**错误代码：**
```cpp
void handleServo() {
  targetAngle = 90;
  while (currentAngle != targetAngle) {  // ❌ 阻塞等待
    updateServo();
    delay(15);
  }
  server.send(200, "text/plain", "OK");  // 延迟几秒才返回
}
```

**正确代码：**
```cpp
void handleServo() {
  targetAngle = 90;
  server.send(200, "text/plain", "OK");  // ✅ 立即返回
}

void loop() {
  server.handleClient();  // 处理请求
  updateServo();          // 舵机慢慢转动
}
```

**参考教程：** [05 网页控制舵机 - 非阻塞设计](05_web_servo_guide.md#332-esp32-立即返回响应)

## 电机通信类问题

### Q12: 电机能通信但不转

**现象：** 串口显示收到回复，但电机不动。

**原因：**
1. 电机处于关闭状态
2. 电机有保护错误
3. 速度或位置目标过小

**解决步骤：**

1. **先发送运行命令**
   
   串口输入：`r` (Run)
   
   确认电机进入运行状态。

2. **检查是否有保护错误**
   
   串口输入：`s` (Status)
   
   查看状态信息，如果有错误：
   - 排除故障原因（如机械卡死、过流）
   - 串口输入：`c` (Clear Error)

3. **增大速度或位置目标**
   
   速度模式：
   ```
   输入 30 或 60（输出轴 dps）
   ```
   
   位置模式：
   ```
   输入 360（输出轴转一圈）
   ```

4. **检查机械负载**
   
   拆掉机械负载，空载测试是否能转。

**参考教程：** [08 电机指南 - 常见问题](08_motor_guide.md#72-串口能发送但电机不动)

---

### Q13: 两个电机ID相同怎么办

**现象：** 最终整合时，两个电机同时响应同一条命令。

**解决方法：**

**方法 1：用厂家上位机修改 ID（推荐）**

参考电机手册，用厂家提供的上位机软件，将两个电机分别设置为 ID 1 和 ID 2。

**方法 2：单独接线，用代码修改**

1. 只接一个电机到 RS485 总线
2. 上传修改 ID 的代码：
   ```cpp
   // 将 ID 1 改为 ID 2
   uint8_t data[] = {0x02};  // 新 ID
   sendCommand(0x???, data, 1);  // 参考厂家协议手册
   ```
3. 断电，接上第二个电机
4. 验证两个电机 ID 不同

**重要：** 修改 ID 后需要断电重启电机。

**参考教程：** [09 机器人整合 - 电机ID要求](09_robot_integration.md#3-电机-id-要求)

## WiFi网页控制类问题

### Q14: 手机搜不到 ESP32 热点

**现象：** 手机 WiFi 列表中看不到 ESP32 热点。

**解决步骤：**

1. **检查串口输出**
   
   打开串口监视器，应该看到：
   ```
   AP IP: 192.168.4.1
   HTTP server started
   ```
   
   如果没有，说明 WiFi 未启动。

2. **检查代码**
   
   确认有：
   ```cpp
   WiFi.softAP("ESP32-Hotspot", "12345678");
   ```

3. **检查密码长度**
   
   密码必须 **至少 8 位**：
   ```cpp
   WiFi.softAP("ESP32", "12345678");  // ✅ 8位，可以
   WiFi.softAP("ESP32", "1234567");   // ❌ 7位，失败
   ```

4. **按 EN 键复位**
   
   上传代码后，按一下 EN 键让程序重新运行。

5. **检查手机设置**
   
   - 某些手机默认隐藏无互联网的热点
   - 临时关闭"移动数据"再搜索
   - 或在 WiFi 设置中手动添加网络

---

### Q15: 连上热点但无法访问网页

**现象：** 手机连接到 ESP32 热点，但浏览器打不开 `192.168.4.1`。

**解决步骤：**

1. **确认 IP 地址**
   
   输入：`http://192.168.4.1`
   
   **不是** `https://192.168.4.1`（没有 s）

2. **关闭 HTTPS 自动跳转**
   
   某些浏览器会自动把 http 改成 https，导致无法访问。
   
   解决方法：
   - 使用不同的浏览器（如 Chrome、Safari）
   - 清除浏览器缓存

3. **检查串口输出**
   
   确认服务器已启动：
   ```
   HTTP server started
   ```

4. **尝试直接访问 IP**
   
   不带 http://，直接输入：`192.168.4.1`

5. **检查防火墙**
   
   某些手机的安全软件会阻止访问本地 IP。

### Q16: 网页按钮点击无反应

**现象：** 网页能打开，但点击按钮后没有反应。

**排查步骤：**

1. **打开浏览器开发者工具**
   
   - 手机：使用电脑浏览器访问（连同一热点）
   - 电脑：按 F12，查看 Console 标签

2. **查看错误信息**
   
   可能看到：
   ```javascript
   Failed to fetch  ← 网络错误
   404 Not Found    ← 路径错误
   ```

3. **检查 ESP32 串口输出**
   
   点击按钮后，串口应该有输出：
   ```
   收到请求: led?state=on
   ```
   
   如果没有，说明请求未到达 ESP32。

4. **检查路由注册**
   
   ESP32 代码中确认：
   ```cpp
   server.on("/led", handleLED);  // 路径和函数对应
   ```

5. **检查 JavaScript 函数**
   
   HTML 中确认：
   ```html
   <button onclick="setLED('on')">开灯</button>
   <script>
   function setLED(state) {
     fetch('/led?state=' + state);
   }
   </script>
   ```

## 串口调试类问题

### Q17: 串口监视器显示乱码

**现象：**
```
������ÿ����������ÿ��
```

**原因：** 波特率不匹配。

**解决方法：**

1. **检查代码中的波特率**
   ```cpp
   Serial.begin(115200);  // 代码设置为 115200
   ```

2. **设置串口监视器波特率**
   
   串口监视器右下角 → 选择 **115200**

3. **按 EN 键复位**
   
   修改波特率后，按 EN 键让程序重新运行。

**常用波特率：**
- 串口调试：`115200`
- RS485 电机：`115200`（默认）
- 低速通信：`9600`

---

### Q18: 串口监视器无输出

**现象：** 打开串口监视器，一片空白，没有任何输出。

**解决步骤：**

1. **检查是否选择了正确的串口**
   
   Tools → Port → 选择实际 COM 口

2. **检查 USB CDC On Boot**
   
   ```
   Tools → USB CDC On Boot → Enabled
   ```
   
   修改后需要重新上传代码。

3. **检查代码中是否有 Serial.begin()**
   
   ```cpp
   void setup() {
     Serial.begin(115200);  // 必须有这一行
   }
   ```

4. **按 EN 键复位**
   
   让程序重新运行。

5. **检查串口是否被占用**
   
   关闭其他串口助手、PuTTY 等程序。
