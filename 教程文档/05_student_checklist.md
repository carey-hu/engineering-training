# 05 项目上机自查清单

## 硬件

- [ ] ESP32-S3 开发板能被电脑识别为 COM 口
- [ ] USB 线确认是数据线，不是只能充电的线
- [ ] 已打开 ESP32 开发板资料页，能查到引脚定义和注意事项
- [ ] 外接 LED、220Ω 电阻、杜邦线已经准备好
- [ ] 外接 LED 已按教程接到 GPIO2，确认不是接到板载 RGB 的 GPIO48
- [ ] RS485 模块已按教程接到 GPIO17、GPIO18、GPIO16
- [ ] 四个舵机信号线已分别接到 GPIO10、GPIO11、GPIO12、GPIO13
- [ ] 控制舵机时使用独立 5V 电源，并确认 ESP32 GND 与舵机电源 GND 共地
- [ ] 控制 MG4010 电机时使用 12V~24V 独立电源、RS485 模块，并确认共地
- [ ] 最终工程使用的两个 MG4010-i10 电机已设置为不同 ID，例如 1 和 2
- [ ] 四个 PWM 舵机已分别确认可单独动作，供电容量足够
- [ ] ESP32、RS485 模块、电机电源负极、舵机电源负极已经连接到公共地

## 软件

- [ ] Arduino IDE 已安装
- [ ] CH340/CH343 驱动已安装或开发板已能自动识别
- [ ] ESP32-S3 板卡包安装成功
- [ ] `示例代码/01_LED闪烁/` 能编译上传
- [ ] 串口监视器能看到 `Serial.println()` 输出
- [ ] Wi-Fi AP 网页控制示例能在浏览器中打开
- [ ] `示例代码/10_机器人整合控制/` 已完成空载测试

## 常见问题自查

- [ ] 无 COM 口：换线、装驱动、看设备管理器
- [ ] 上传失败：BOOT 键、降速到 115200
- [ ] 串口无输出：USB CDC On Boot 开启
- [ ] 编译失败：检查中文路径、板卡包版本
- [ ] 网页控制卡顿：检查 delay、滑块请求频率
- [ ] 电机无回复：检查 115200 波特率、ID=1、A/B 接线、DE/RE 和共地
- [ ] 第二个电机无回复：检查 ID 是否为 2，两个电机是否误设成同一 ID
- [ ] 电机不动：先发送运行命令，低速小角度测试，确认没有保护错误
- [ ] 四舵机同时抖动：优先检查 5V 舵机电源容量和共地，不要从 ESP32 取电

## 项目开始前的最低要求

- [ ] 能独立上传 LED 闪烁程序
- [ ] 能看懂开发板上的 3.3V、5V、GND 和 GPIO 标注
- [ ] 能说明 5V-IN 默认不向外供电，不能直接当作舵机电源
- [ ] 能说明为什么舵机和电机不能由 ESP32 的 3.3V 引脚供电
- [ ] 能说明 RS485 的 A/B、UART 的 TX/RX、DE/RE 分别是什么
- [ ] 能在异常声音、发热、复位或失控时立即断电
