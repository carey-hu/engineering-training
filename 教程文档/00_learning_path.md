## 0. 开始前先确认

### 你需要准备

| 物品 | 用途 |
|------|------|
| ESP32-S3 开发板 | 主控制器 |
| USB 数据线 | 上传程序和串口调试，不能只用充电线 |
| 板载普通 LED | 开发板自带，用来做第一次无接线 GPIO 输出实验 |
| 普通 PWM 舵机 | 学习 50Hz PWM 控制 |
| 独立 5V 舵机电源 | 给舵机供电 |
| MG4010-i10 RS485 电机 | 学习 RS485 电机控制 |
| 12V~24V 电机电源 | 给电机供电 |
| RS485/供电转接板 | 推荐使用，连接 ESP32、电机电源和 MG4010 电机 |
| 3.3V 兼容 RS485 模块 | 没有转接板时的替代方案 |

### 安全底线

> [!CAUTION]
> 上电前务必确认以下四条，否则可能烧板、烧电源或失控伤人：
> 1. 舵机和电机都不能从 ESP32 的 3.3V 引脚取电。
> 2. ESP32 GND、RS485/供电转接板 GND、电机电源负极、舵机电源负极最终必须共地。
> 3. 第一次测试电机时先低速、小角度、空载。
> 4. 出现发热、异响、复位、失控时立即断电。

## 1. 环境搭建

### 相关教程

- Windows：[01_install_windows.md](01_install_windows.md)
- macOS：[02_install_macos.md](02_install_macos.md)
- 下载地址：[00_downloads_cn.md](00_downloads_cn.md)
- 板卡参数：[03_board_settings_esp32s3.md](03_board_settings_esp32s3.md)

### 要做的事

1. 安装 Arduino IDE 2.x。
2. 添加 ESP32 国内镜像地址。
3. 安装带 `-cn` 后缀的 `esp32 by Espressif Systems` 板卡包。
4. 选择 `ESP32S3 Dev Module` 和正确 COM 口。
5. 开启 `USB CDC On Boot: Enabled`。

### 完成标志

- [x] Arduino IDE 能看到 ESP32-S3 板卡
- [x] 插拔开发板时能看到 COM 口变化
- [x] 串口监视器能设置为 `115200`

### 常见问题

| 现象 | 处理 |
|------|------|
| 找不到 COM 口 | 先换 USB 数据线和 USB 口，再检查 Windows 设备管理器中的端口变化 |
| 板卡包装不上 | 确认用的是 `package_esp32_index_cn.json` 国内镜像地址，并安装带 `-cn` 后缀的版本 |
| 上传卡住 | 降低 Upload Speed 到 `115200`，按住 BOOT 再上传 |

## 5. RS485 电机：先读状态，再运行

### 相关教程

- [08_motor_guide.md](08_motor_guide.md)
- 电机协议 PDF：[硬件资料/技术文档/MG4010-i10-R485协议.pdf](../硬件资料/技术文档/MG4010-i10-R485协议.pdf)

### 运行顺序

| 顺序 | 示例 | 目标 |
|------|------|------|
| 1 | [06_电机基础控制](../示例代码/06_电机基础控制/06_mg4010_motor_basic.ino) | 运行、停止、读状态、读角度 |
| 2 | [07_电机速度模式](../示例代码/07_电机速度模式/07_motor_speed_mode.ino) | 输入输出轴速度，低速正反转 |
| 3 | [08_电机位置模式](../示例代码/08_电机位置模式/08_motor_position_mode.ino) | 输入输出轴角度，低速到达目标位置 |
| 4 | [09_电机网页控制](../示例代码/09_电机网页控制/09_motor_web_control.ino) | 用网页切换速度/位置模式 |

### 接线说明

> [!TIP]
> 推荐使用课程配套 RS485/供电转接板。转接板把 RS485 收发器、电机电源端子和电机 A/B 总线集中到一块板上，能减少学生把电源、GND、A/B 接散的错误。原理图见：[硬件资料/技术文档/转接板原理图.pdf](../硬件资料/技术文档/转接板原理图.pdf)。

#### 使用转接板时

先按板子丝印接线，重点核对：

```text
ESP32 5V         -> 转接板 USB调试 5V
ESP32 GND        -> 转接板 USB调试 G
ESP32 GPIO17(TX) -> 转接板 TTL-485 调试 T
ESP32 GPIO18(RX) -> 转接板 TTL-485 调试 R
24V 电源正极     -> 转接板输入端子上孔
24V 电源负极     -> 转接板输入端子下孔
电机 6pin        -> 从左到右 B、A、GND、GND、24V、24V
```

> [!NOTE]
> 课程配套转接板已经在板上处理 RS485 收发方向，ESP32 不需要再接 GPIO16。

#### 使用独立 RS485 模块时

```text
GPIO17 -> RS485 DI
GPIO18 -> RS485 RO
GPIO16 -> RS485 DE 和 RE
ESP32 GND -> RS485 GND -> 电机电源 V-
RS485 A -> 电机 A/H
RS485 B -> 电机 B/L
```

### 首次测试顺序

1. 断电检查 V+、V-、A/B、GND。
2. 上传 `06_电机基础控制`。
3. 打开串口监视器。
4. 给电机上电。
5. 先输入 `s` 读状态。
6. 再输入 `r` 运行。
7. 输入 `t` 停止。

### 完成标志

- [x] `s` 能读到温度、速度、编码器
- [x] `a` 能读到多圈角度
- [x] 速度模式输入 `30` 或 `-30` 时，输出轴能低速正反转

### 常见问题

| 现象 | 处理 |
|------|------|
| 没有回复 | 使用转接板时按 ID、波特率、TX/RX、A/B、共地顺序排查；使用通用 RS485 模块时再检查 DE/RE |
| 有回复但不动 | 先发 `r` 运行命令，确认没有保护错误 |
| 方向相反 | 先用正负号处理，不要急着改电机内部参数 |

## 6. 最终整合：两个电机 + 四个舵机

### 相关教程

- [09_robot_integration.md](09_robot_integration.md)
- [示例代码/10_机器人整合控制/README.md](../示例代码/10_机器人整合控制/README.md)

### 对应代码

[示例代码/10_机器人整合控制/10_robot_integrated_control.ino](../示例代码/10_机器人整合控制/10_robot_integrated_control.ino)

### 调试顺序

1. 只接四个舵机，测试 `s 1 90`、`pose 90 90 90 90`。
2. 只接两个电机，确认 ID 分别为 1 和 2，测试 `v 30 30`。
3. 舵机和电机都接上，空载低速测试。
4. 装到机器人结构后，再逐步增加速度和动作范围。

### 串口命令

| 命令 | 作用 |
|------|------|
| `v 30 30` | 左右电机输出轴速度设为 30 dps，程序会自动发送运行命令 |
| `v 30 -30` | 左右电机反向，适合原地转向测试 |
| `stop` | 两个电机停止 |
| `s 1 90` | 舵机 1 转到 90° |
| `pose 90 90 90 90` | 四个舵机分别转到指定角度 |
| `home` | 电机速度归零，四个舵机回中 |

### 完成标志

- [x] 四个舵机可单独控制
- [x] 两个电机能分别按 ID 响应
- [x] 舵机动作时电机通信不明显丢包
- [x] 电机动作时 ESP32 不复位

## 8. 最后确认
