# 06 硬件资料说明

> 这篇文档帮你搞清楚开发板的硬件细节、原理图在哪、驱动怎么装、3D 图纸是干啥用的。

---

## 1. 开发板详细规格

### 核心芯片：ESP32-S3

这是乐鑫（Espressif）出品的一颗双核处理器，专门为物联网（IoT）应用设计的。

**主要参数：**

```text
CPU：双核 Xtensa LX7，最高 240MHz
RAM：512KB SRAM（可选外挂 PSRAM，常见 2MB/8MB）
Flash：通常 8MB 或 16MB（看具体开发板配置）
Wi-Fi：支持 2.4GHz 802.11 b/g/n
Bluetooth：支持 Bluetooth 5 (LE)
USB：原生 USB OTG 1.1（全速 12Mbps）
GPIO：最多 45 个可编程 GPIO 引脚
外设：SPI、I2C、UART、PWM、ADC、DAC、触摸传感器等
```

**简单说就是**：比 ESP8266 强多了，比 ESP32 又多了原生 USB，适合做智能硬件、机器人、传感器项目。

---

### 开发板常见配置（以新陆城 xinlucity 开发板为例）

本课程使用的 ESP32-S3 开发板通常会包含以下配置：

```text
芯片：ESP32-S3-WROOM-1 或 ESP32-S3-WROOM-2 模组
Flash：8MB 或 16MB
PSRAM：2MB 或 8MB（部分型号）
USB 转串口芯片：CH340G / CH343P（部分型号）
原生 USB：支持（通过 ESP32-S3 芯片自带的 USB 功能）
按键：BOOT + EN（复位键）
LED：板载电源指示灯 + 可编程 LED（GPIO 位置看具体型号）
引脚引出：双排排针，方便面包板插接
供电：USB 供电（5V），板上 3.3V 稳压输出
尺寸：约 55mm × 28mm（标准开发板尺寸）
```

**注意事项：**

- 不同批次的开发板，板载 LED 引脚可能不同（有的是 GPIO2，有的是 GPIO48）
- 部分开发板没有外置 USB 转串口芯片，直接用 ESP32-S3 的原生 USB
- 引脚丝印（引脚标注）要看清楚，别接错 3.3V 和 GND

---

### 开发板上的关键部件

#### USB 口

```text
作用：供电 + 代码上传 + 串口通信
注意：必须用数据线，充电线不行
```

**两种 USB 模式：**

1. **原生 USB 模式**：ESP32-S3 芯片自己提供 USB 功能，设备管理器里会显示 `USB JTAG/serial debug unit` 或类似名称
2. **外置串口芯片模式**：板子上有 CH340/CH343 芯片，设备管理器里会显示 `USB-SERIAL CH340 (COMx)`

大多数新款开发板支持原生 USB，驱动更方便。

---

#### BOOT 按键

```text
作用：强制进入下载模式
用法：按住 BOOT → 点上传 → 看到 Connecting... → 松开 BOOT
```

正常情况下不需要按，但如果上传卡住，这是最常用的救命操作。

---

#### EN 按键（也叫 RST 或 RESET）

```text
作用：复位开发板
用法：按一下，程序从头开始运行
```

调试的时候经常用到，相当于重启电脑。

---

#### 电源指示灯

```text
通常是红色或蓝色，插上 USB 就亮
如果不亮，检查 USB 线和接口
```

---

#### GPIO 引脚（排针）

```text
作用：连接传感器、电机、LED、按钮等外部设备
注意：电压是 3.3V，别直接接 5V 设备
```

常用引脚类型：

```text
GND：地线（负极）
3.3V：电源输出（正极）
5V：USB 供电输出（部分板子有）
GPIO0-GPIO48：可编程数字引脚
ADC1/ADC2：模拟输入引脚（读传感器电压）
TX/RX：串口通信引脚
SDA/SCL：I2C 通信引脚
MOSI/MISO/SCK：SPI 通信引脚
```

## 2. 原理图位置

原理图文件在硬件资料目录中：

[ESP32-S3原理图.pdf](../硬件资料/技术文档/ESP32-S3原理图.pdf)

**原理图是干啥的？**

原理图（Schematic）就是开发板的"电路设计图纸"，告诉你：

- 每个引脚连到哪个芯片
- 芯片之间怎么连接的
- 电阻、电容、电源芯片的参数
- USB、按键、LED 的电路设计

**什么时候需要看原理图？**

```text
- 想知道板载 LED 连在哪个 GPIO
- 搞清楚 BOOT 和 EN 按键的电路原理
- 确认开发板支不支持某个功能（比如 PSRAM）
- 调试硬件问题（比如引脚冲突）
- 自己做 PCB 时参考设计
```

**初学者需要看吗？**

不用。跟着教程用就行，遇到问题再翻。

## 3. 驱动安装指引

### 什么时候需要装驱动？

插上开发板后，打开设备管理器：

```text
Windows：右键"此电脑" → 管理 → 设备管理器
macOS：系统报告 → USB
```

情况 1：出现 COM 口（正常）

```text
端口（COM 和 LPT）
  └─ USB-SERIAL CH340 (COM3)
```

说明驱动已经装好了，不用管。

---

情况 2：出现未知设备（需要装驱动）

```text
其他设备
  └─ USB2.0-Serial（带黄色感叹号）
```

说明驱动没装，需要手动装。

---

情况 3：啥也没出现（检查硬件）

检查：

```text
1. USB 线是不是数据线（换根线试试）
2. 开发板灯亮了吗（没亮说明供电有问题）
3. 换个 USB 口试试
4. 按住 BOOT 键再插 USB
```

---

### 驱动安装步骤（Windows）

#### CH340/CH341 驱动

**适用对象：** 板子上有 CH340G 或 CH341 芯片的开发板

**下载地址：**

- 官方：https://www.wch.cn/downloads/CH341SER_EXE.html
- 本地：[硬件资料/驱动程序/CH341SER.EXE](../硬件资料/驱动程序/CH341SER.EXE)

**安装步骤：**

```text
1. 双击 CH341SER.EXE
2. 点"安装"
3. 等待安装完成
4. 重新插拔 USB 线
5. 设备管理器里应该能看到 COM 口了
```

---

#### CH343/CH342/CH344 驱动

**适用对象：** 板子上有 CH343P 等新型号 WCH 串口芯片

**下载地址：**

```text
官方：https://www.wch.cn/downloads/CH343SER_EXE.html
```

**安装步骤：** 和 CH340 一样，双击安装包，点安装，重新插拔 USB。

---

#### 原生 USB 模式（不需要装驱动）

如果你的开发板用的是 ESP32-S3 原生 USB，设备管理器里会显示：

```text
通用串行总线设备
  └─ USB JTAG/serial debug unit
```

或者：

```text
端口（COM 和 LPT）
  └─ USB Serial Device (COMx)
```

**Windows 10/11 自动识别，不需要装驱动。**

---

### 驱动安装（macOS）

**CH340/CH341 驱动：**

```text
下载：https://www.wch.cn/downloads/ch34xser_mac_zip.html
安装后，如果系统提示"已阻止"，去：
  系统设置 → 隐私与安全性 → 允许"江苏沁恒股份有限公司"
然后重启 Mac。
```

**原生 USB 模式：**

macOS 自动识别，不需要装驱动。

---

### 驱动问题排查

#### 问题 1：装了驱动还是没 COM 口

**可能原因：**

```text
1. 驱动装错版本了（CH340 和 CH343 是不同的驱动）
2. USB 线是充电线，不是数据线
3. 开发板坏了或接触不良
4. Windows 驱动被杀毒软件拦截了
```

**解决办法：**

```text
1. 确认板子上的串口芯片型号（看板子上的小芯片，通常印有 CH340G 或 CH343P）
2. 卸载旧驱动，重新装对应版本
3. 换根 USB 数据线
4. 换台电脑试试
```

---

#### 问题 2：COM 口有了，上传还是失败

**检查项：**

```text
- Arduino IDE 里选对 COM 口了吗？
- 选对开发板型号了吗？（Tools → Board → ESP32S3 Dev Module）
- Upload Speed 是不是太快？（试试改成 115200）
- 串口监视器是不是开着？（关掉再上传）
```

---

#### 问题 3：macOS 驱动装了还是不认

**可能原因：**

```text
macOS 对第三方内核扩展管控比较严
```

**解决办法：**

```text
1. 系统设置 → 隐私与安全性 → 允许驱动扩展
2. 重启 Mac
3. 如果还是不行，考虑换一块支持原生 USB 的开发板
```

## 4. 3D 图纸用途

硬件资料目录中提供了 3D 模型文件：

[MG4010-i10电机.STEP](../硬件资料/3D模型/电机/MG4010-i10电机.STEP)

这是 **MG4010 电机的 3D 图纸**，不是开发板的。

### STEP 文件是什么？

STEP（.step 或 .stp）是一种通用的 3D 模型格式，可以用来：

```text
- 在 CAD 软件里查看电机尺寸
- 设计机器人时做装配仿真
- 3D 打印电机支架或外壳
- 检查电机和其他零件的安装间隙
```

### 怎么打开 STEP 文件？

**免费软件：**

```text
FreeCAD：https://www.freecad.org/
Fusion 360（学生免费）：https://www.autodesk.com/products/fusion-360/
Onshape（网页版免费）：https://www.onshape.com/
```

**商业软件：**

```text
SolidWorks
AutoCAD
Inventor
```

### 什么时候用得上？

```text
- 你要做一个小车或机械臂，需要给电机设计安装支架
- 需要确认电机能不能装进你设计的外壳里
- 要做结构仿真，检查电机和其他部件会不会碰撞
```

**如果你只是写代码控制电机，不需要打开这个文件。**

## 6. 常见硬件问题

### 问题 1：GPIO 引脚接了设备，开发板启动不了

**原因：** 某些引脚是启动引脚（Strapping Pins），上电时电平状态会影响启动模式。

**避免使用的引脚：**

```text
GPIO0：启动模式选择
GPIO3：JTAG 引脚
GPIO45：VDD_SPI 电压选择
GPIO46：启动模式选择
```

**建议：** 用 GPIO2、GPIO4、GPIO5 等普通 GPIO 做实验。

---

### 问题 2：5V 设备能直接接 ESP32-S3 的 GPIO 吗？

**不能！** ESP32-S3 的 GPIO 电压是 3.3V，接 5V 设备会烧坏引脚。

**解决办法：**

```text
1. 用电平转换模块（如 TXS0108E）
2. 用电阻分压（简单但不太准）
3. 选 3.3V 兼容的传感器和模块
```

---

### 问题 3：板载 LED 不亮

**检查项：**

```text
1. 代码里的引脚号对吗？（不同板子 LED 引脚不同）
2. pinMode 设置对了吗？（应该是 OUTPUT）
3. digitalWrite 写的是 HIGH 还是 LOW？（有些板子 LED 是低电平点亮）
4. 板子上真的有可编程 LED 吗？（有些板子只有电源指示灯）
```

**调试方法：**

```text
用万用表测 GPIO 引脚电压
或者接外部 LED 测试（GPIO → 220Ω电阻 → LED 长脚 → LED 短脚 → GND）
```

---

### 问题 4：开发板发烫

**正常情况：**

```text
ESP32-S3 工作时会发热，尤其是跑 Wi-Fi 或高频运算时
温度在 40-60°C 是正常的
```

**不正常情况：**

```text
烫手（超过 80°C）
闻到焦味
板子冒烟
```

**可能原因：**

```text
1. 短路了（检查接线）
2. 供电电压太高（标准是 5V）
3. GPIO 输出接了过大负载（比如直接接电机）
```

**解决办法：**

```text
立即断电，检查电路，必要时换块新板子
```

## 7. 项目硬件核对（给教师）

本课程默认开发板、电机、舵机和基础配件已经准备完成。课前重点不是再讨论硬件选型，而是确认硬件能否支撑最终机器人项目。

### 开发板核对

```text
- 能被电脑识别为 COM 口
- Arduino IDE 能完成 LED 闪烁上传
- GPIO 丝印或引脚图清楚，便于学生按图接线
- 至少保留 7 个可用 GPIO：3 个给 RS485，4 个给 PWM 舵机
- 供电和 GND 引脚位置明确，避免学生误接
```

### 最终工程资源占用

| 功能 | ESP32-S3 资源 | 本教程默认引脚 |
|------|---------------|----------------|
| RS485 TX | UART2 TX | GPIO17 |
| RS485 RX | UART2 RX | GPIO18 |
| RS485 DE/RE | 普通数字输出 | GPIO16 |
| 舵机 1 | LEDC PWM | GPIO10 |
| 舵机 2 | LEDC PWM | GPIO11 |
| 舵机 3 | LEDC PWM | GPIO12 |
| 舵机 4 | LEDC PWM | GPIO13 |

如果实际开发板没有引出这些 GPIO，可以在代码顶部修改引脚定义。不要使用板子上已经接到特殊外设、启动模式或不可用焊盘的引脚。

### 上课前必须完成的检查

```text
1. USB 数据线能上传程序，不是只能充电的线。
2. ESP32-S3、RS485 模块、电机电源负极、舵机电源负极已经共地。
3. 两个 MG4010-i10 电机已经设置为不同 ID，例如 1 和 2。
4. 舵机 5V 电源容量足够，不能从 ESP32 3.3V 引脚取电。
5. 电机电源和舵机电源分别接线清楚，学生不会把电机接到开发板供电引脚。
```

这篇文档整理了开发板的硬件规格、原理图位置、驱动安装方法和 3D 图纸用途。

**记住这几点：**

```text
1. 开发板核心是 ESP32-S3 芯片，双核 240MHz，支持 Wi-Fi 和蓝牙

5. GPIO 电压是 3.3V，别接 5V 设备
6. 有问题先检查：USB 线、COM 口、驱动、接线
```
