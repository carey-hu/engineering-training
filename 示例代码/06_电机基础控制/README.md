# MG4010 电机基础控制（RS485 自定义协议）

## 功能说明

通过 RS485 串口控制 MG4010 无刷电机，使用自定义协议实现电机的启停和速度控制。

## 硬件需求

1. **ESP32-S3 开发板**
2. **MG4010 无刷电机及驱动器**
3. **RS485 转换模块**（MAX485 或 MAX3485）
4. **24V 直流电源**（用于电机驱动）
5. **连接线材**（杜邦线、RS485 双绞线）

## 接线说明

### 完整接线图

```
┌─────────────────┐         ┌──────────────────┐         ┌─────────────────┐
│   ESP32-S3      │         │   RS485 模块     │         │   MG4010 电机   │
│                 │         │  (MAX485/3485)   │         │     驱动器      │
├─────────────────┤         ├──────────────────┤         ├─────────────────┤
│ GPIO17 (TX2) ───┼────────►│ DI (TXD)         │         │                 │
│ GPIO18 (RX2) ───┼────────►│ RO (RXD)         │         │                 │
│ GPIO16 (DE)  ───┼────────►│ DE               │   RS485 │  A+  (红/绿)    │
│ GPIO16 (RE)  ───┼────────►│ RE               │   双绞线│  B-  (蓝/白)    │
│                 │         │ A+  ─────────────┼─────────┤                 │
│ 3.3V ───────────┼────────►│ VCC              │  (屏蔽) │  GND            │
│ GND  ───────────┼─────┬──►│ B-  ─────────────┼─────────┤                 │
│                 │     │   │ GND              │         │                 │
└─────────────────┘     │   └──────────────────┘         │  DC 24V+        │
                        │                                │  DC 24V-        │
                        │                                └─────────────────┘
                        │                                        │
                        └────────────────────────────────────────┘
                                    共地连接必须！
```

### 详细引脚连接

#### ESP32-S3 → RS485 模块

| ESP32-S3 引脚 | RS485 模块引脚 | 说明 |
|--------------|---------------|------|
| GPIO17 (TX2) | DI (TXD)      | 发送数据 |
| GPIO18 (RX2) | RO (RXD)      | 接收数据 |
| GPIO16       | DE + RE       | 收发控制（两个引脚连在一起） |
| 3.3V         | VCC           | 电源（部分模块需要 5V，查看规格书） |
| GND          | GND           | 地线 |

#### RS485 模块 → MG4010 驱动器

| RS485 模块 | MG4010 驱动器 | 线缆颜色参考 |
|-----------|--------------|-------------|
| A+        | A+ (485+)    | 红色或绿色   |
| B-        | B- (485-)    | 蓝色或白色   |

#### 电源连接

| 设备 | 电源 | 注意事项 |
|------|------|---------|
| ESP32-S3 | USB 供电或 5V | 调试时使用 USB |
| RS485 模块 | 3.3V 或 5V | 查看模块规格 |
| MG4010 驱动 | 24V DC 电源 | **必须使用外部电源** |

**重要：必须将 ESP32-S3 的 GND 和 MG4010 驱动器的 GND 连接在一起（共地）！**

## RS485 模块说明

### 引脚功能

- **DI (Driver Input)**：数据输入，连接到 MCU 的 TX
- **RO (Receiver Output)**：数据输出，连接到 MCU 的 RX
- **DE (Driver Enable)**：发送使能，高电平时发送
- **RE (Receiver Enable)**：接收使能，低电平时接收
- **VCC**：电源（3.3V 或 5V）
- **GND**：地线
- **A+**：RS485 差分信号正极
- **B-**：RS485 差分信号负极

### 常见 RS485 模块

1. **基于 MAX485**：最常见，价格便宜
2. **基于 MAX3485**：增强型，抗干扰能力更强
3. **自动方向控制型**：有些模块内置自动收发切换，不需要 DE/RE 控制

## 协议说明

### 自定义协议格式

```
| 帧头 | 地址 | 功能码 | 数据长度 | 数据 | CRC16 低字节 | CRC16 高字节 |
| 0xAA | 0x01 |  0xXX  |   0xXX   | [...] |     0xXX     |     0xXX     |
```

### 功能码定义

| 功能码 | 命令 | 数据 | 说明 |
|-------|------|------|------|
| 0x01  | 启动电机 | 0x01 | 启动电机运行 |
| 0x02  | 停止电机 | 0x00 | 停止电机运行 |
| 0x03  | 设置速度 | 速度值(0-100) | 设置电机转速百分比 |
| 0x04  | 读取状态 | 无 | 查询电机当前状态 |

### 数据格式示例

**启动电机命令：**
```
0xAA 0x01 0x01 0x01 0x01 [CRC_L] [CRC_H]
```

**设置速度 50% 命令：**
```
0xAA 0x01 0x03 0x02 0x00 0x32 [CRC_L] [CRC_H]
```

## 使用步骤

### 1. 硬件连接

1. 按照接线图连接所有硬件
2. **确保共地连接**（ESP32 GND - RS485 GND - 电机驱动 GND）
3. 连接 24V 电源到电机驱动器
4. 检查 RS485 的 A+ 和 B- 连接是否正确

### 2. 软件配置

1. 打开 Arduino IDE
2. 选择开发板：`ESP32S3 Dev Module`
3. 确认配置：
   - USB CDC On Boot: **Enabled**
   - Upload Speed: 921600
   - USB Mode: Hardware CDC and JTAG
4. 选择正确的端口
5. 上传程序

### 3. 测试运行

1. 打开串口监视器（波特率 115200）
2. 应该看到初始化信息和帮助菜单
3. 输入 `h` 查看命令列表
4. 输入 `s` 启动电机
5. 输入 `5` 设置速度为 50%
6. 输入 `+` 或 `-` 调整速度
7. 输入 `p` 停止电机

## 注意事项

### 安全警告

1. **电源隔离**：ESP32 使用 USB 或 5V 供电，电机驱动使用 24V 电源，两者电源不能混用
2. **共地连接**：必须将所有设备的 GND 连接在一起
3. **接线检查**：上电前仔细检查 A+/B- 是否接反
4. **电机固定**：测试时确保电机固定牢固，防止飞转
5. **急停准备**：测试时准备好切断电源的方法

### 调试技巧

1. **串口无输出**：
   - 检查 USB CDC On Boot 是否启用
   - 按下 EN/RST 按钮复位
   - 检查串口波特率是否为 115200

2. **电机无响应**：
   - 检查 RS485 A+/B- 是否接反
   - 用万用表测量 24V 电源是否正常
   - 检查共地连接
   - 串口监视器查看发送的命令数据

3. **通信错误**：
   - 检查 DE/RE 引脚是否正确控制
   - 降低波特率重试（改为 4800）
   - 检查双绞线长度（建议 < 10m 测试）

4. **查看原始数据**：
   - 程序中已包含十六进制数据输出
   - 可以用逻辑分析仪抓取 RS485 信号

### 终端电阻

如果通信距离较远（> 10m）或有多个设备，需要在 RS485 总线两端加 120Ω 终端电阻：

```
A+ ──┬── [120Ω] ──┬── A+
     │            │
B- ──┴────────────┴── B-
  (起点)        (终点)
```

## 扩展功能

### 1. 多电机控制

修改 `MOTOR_ADDR` 为不同地址，可以控制多个电机：

```cpp
#define MOTOR_ADDR_1    0x01
#define MOTOR_ADDR_2    0x02
#define MOTOR_ADDR_3    0x03
```

### 2. 加速度控制

添加平滑加减速功能，避免突然启停：

```cpp
void smoothSetSpeed(int targetSpeed, int stepDelay) {
  while (currentSpeed != targetSpeed) {
    if (currentSpeed < targetSpeed) {
      currentSpeed++;
    } else {
      currentSpeed--;
    }
    setMotorSpeed(currentSpeed);
    delay(stepDelay);
  }
}
```

### 3. 状态监测

周期性读取电机状态（转速、电流、温度等）：

```cpp
void loop() {
  static unsigned long lastCheck = 0;
  if (millis() - lastCheck > 1000) {
    readMotorStatus();
    lastCheck = millis();
  }
}
```

### 4. 故障检测

添加通信超时和故障检测：

```cpp
bool checkMotorCommunication() {
  readMotorStatus();
  // 检查响应
  if (/* 无响应 */) {
    Serial.println("通信故障！");
    return false;
  }
  return true;
}
```

## 常见问题

### Q1: 如何判断 RS485 模块是 3.3V 还是 5V？

**A**: 查看模块上的标识，或查阅卖家提供的说明。大多数模块支持 3.3V-5V 宽电压。使用 3.3V 更安全（ESP32-S3 的 IO 电压为 3.3V）。

### Q2: DE 和 RE 必须连在一起吗？

**A**: 大多数应用中是的。DE 高电平发送，RE 低电平接收，它们的逻辑刚好相反，所以可以共用一个 GPIO 控制。

### Q3: RS485 最大通信距离是多少？

**A**: 理论上 1200m，但实际取决于波特率、线缆质量、干扰环境。建议：
- 9600 bps：500m 以内
- 19200 bps：300m 以内
- 115200 bps：50m 以内

### Q4: 可以用 USB 转 RS485 模块直接连电脑测试吗？

**A**: 可以！使用串口调试工具（如 Serial Port Utility）发送十六进制命令进行测试。

### Q5: MG4010 的协议是标准的吗？

**A**: MG4010 使用**自定义协议**，不是标准 Modbus。本例程提供了基础框架，具体协议细节需要参考 MG4010 的通信协议文档。
