# 机器人整合控制：两个 RS485 电机 + 四个 PWM 舵机

本示例用于最终机器人工程的主程序模板，展示同一块 ESP32-S3 同时控制：

- 2 个 MG4010-i10 RS485 电机，默认 ID 为 1 和 2
- 4 个 PWM 舵机，默认使用 GPIO10、GPIO11、GPIO12、GPIO13

## 接线

课程配套 RS485/供电转接板：

| ESP32-S3 | 转接板 |
|----------|--------|
| 5V | USB调试 5V |
| GND | USB调试 G |
| GPIO17 (TX) | TTL-485 调试 T |
| GPIO18 (RX) | TTL-485 调试 R |

课程配套转接板已在板上处理 RS485 收发方向，ESP32 不需要接 GPIO16。如果改用通用 RS485 模块，GPIO17 接 DI，GPIO18 接 RO，GPIO16 接 DE 和 RE。

电机：

| RS485 转接板/模块 | MG4010-i10 |
|------------|------------|
| 电机 6pin 第 1 pin：B | B/L |
| 电机 6pin 第 2 pin：A | A/H |
| 电机 6pin 第 3、4 pin：GND | V- |
| 电机 6pin 第 5、6 pin：24V | V+ |

舵机信号线：

| 舵机 | ESP32-S3 |
|------|----------|
| 舵机 1 | GPIO10 |
| 舵机 2 | GPIO11 |
| 舵机 3 | GPIO12 |
| 舵机 4 | GPIO13 |

GPIO10、GPIO11、GPIO12、GPIO13 已从排针引出，只接舵机信号线。舵机红线和棕/黑线接独立 5V 舵机电源，不要从 ESP32 3.3V 取电。

供电要求：

- 电机使用独立 12V~24V 电源。
- 舵机使用独立 5V 电源。
- ESP32 GND、RS485/供电转接板或 RS485 模块 GND、电机电源负极、舵机电源负极必须共地。

## 串口命令

打开串口监视器，波特率设为 `115200`，行尾选择“换行”或“Both NL & CR”。

| 命令 | 作用 |
|------|------|
| `run` | 两个电机进入运行状态。速度命令会自动运行电机，这个命令主要用于单独测试 |
| `stop` | 两个电机停止 |
| `v 30 30` | 左右电机输出轴速度设为 30 dps，程序会自动发送运行命令 |
| `v 30 -30` | 左右电机反向，机器人原地转向 |
| `s 1 90` | 舵机 1 转到 90° |
| `pose 90 90 90 90` | 四个舵机分别转到指定角度 |
| `home` | 四个舵机回到 90°，两个电机速度设为 0 |

## 移植说明

前面 03、04、05、06、07、08 例程用于学习单个模块。最终工程建议以本示例为基础：

- 保留非阻塞舵机更新结构。
- 电机命令函数保留 `id` 参数。
- 左右电机速度命令按顺序发送到 ID 1 和 ID 2。
- 网页控制或传感器逻辑应加入主循环调度，不要使用长时间 `delay()`。

## 稳定性说明

本示例已经针对“两电机四舵机同时工作”做了基础处理：

- 舵机 PWM 由 ESP32-S3 LEDC 外设输出，和 UART2 串口不是同一硬件资源。
- 四个舵机采用轮询小步更新，不在同一轮循环里全部大幅改变角度。
- 等待电机 RS485 回包时仍会调用 `updateServos()`，避免串口等待期间舵机调度完全停止。

如果出现四个舵机同时动作时不到位，优先检查舵机 5V 电源容量、共地和机械卡滞。
如果出现舵机动作时电机无回复，优先检查公共地、舵机电源跌落、RS485 A/B 接线和两个电机 ID 是否重复。

建议调试顺序：

1. 只测试四个舵机。
2. 只测试两个 RS485 电机。
3. 空载同时测试四个舵机和两个电机。
4. 安装到机器人结构后低速、小角度测试。
