# GitHub 仓库阅读排版优化建议书（交给 AI 执行）

> 目标仓库：`carey-hu/engineering-training`
> 执行方式：AI 读本文件 → clone 仓库 → 按本文逐文件修改 → 自检 → commit & push（或建 PR）。
> 一句话目标：**不改任何技术内容**，只把 README 和教程文档的**阅读排版与引导逻辑**优化成对零基础学生友好、在 GitHub/Gitee 上直接发链接就能用。

---

## 1. 优化总原则（每个文件都遵守）

1. **第一步顶到最前**：任何页面打开，学生第一眼要看到「现在该做什么」，而不是一堆入口或参数。
2. **线性主线**：用「学习路线 + 上一步/下一步」把零基础学生串成一条路，弥补 GitHub 没有引导式导航的短板。
3. **重点跳出来**：安全、前置条件用 GitHub 警告框（`[!CAUTION]` 等）渲染成醒目色块。
4. **密集资料折叠**：参数表、引脚表、资源清单等"查阅型"内容用 `<details>` 收起，默认不干扰阅读。
5. **统一结构**：所有教程文档用同一套小标题顺序（见 §5），学生形成肌肉记忆。
6. **只重组、不臆造、不删信息**：所有技术细节以仓库原文为准；代码不改逻辑。

## 2. GitHub 原生语法速查（本次要用到的）

**警告框**（GitHub 渲染为彩色块；Gitee 不支持会降级成普通引用，内容不丢）：
```markdown
> [!NOTE] 一般补充说明
> [!TIP] 小技巧 / 提效做法
> [!IMPORTANT] 必须先做 / 关键前置
> [!WARNING] 可能出错的注意点
> [!CAUTION] 危险 / 安全底线（红色）
```

**折叠块**（GitHub + Gitee 都支持）：
```markdown
<details>
<summary><b>标题（点开查看）</b></summary>

<br>

…内容…

</details>
```

**任务清单**（完成标志用）：`- [ ] 未完成` / `- [x] 已完成`

**代码块**：Arduino 用 ` ```cpp `，参数/地址用 ` ```text `。

**相对链接 / 图片**：保持仓库内相对路径，例如 `[舵机指南](教程文档/07_servo_guide.md)`、`![](硬件资料/接线图/xxx.png)`。中文路径在 GitHub/Gitee 链接里可直接用，**不要 URL 编码**。

**锚点**：标题自动生成锚点，可用 `[跳转](#本节目标)` 在长文里做页内 TOC。

## 6. 提示框映射规则（统一口径）

| 原文里的内容 | 改成 |
|------|------|
| 安全底线、共地、独立供电、断电、可能烧板/伤人 | `> [!CAUTION]` |
| 必须先完成 X、不要跳步、关键前置 | `> [!IMPORTANT]` |
| 上传失败应对、国内镜像、提效小技巧 | `> [!TIP]` |
| 可能踩坑但不严重的注意 | `> [!WARNING]` |
| 一般补充、版本差异说明 | `> [!NOTE]` |

同一篇里安全类只保留 1~2 个 `[!CAUTION]`，避免红块泛滥反而被忽略。

## 7. 红线约束（务必遵守）

- **不改 `.ino` 代码逻辑**；不臆造引脚号、命令字（如 `0x88/0x81/0x9C`）、波特率、热点名密码等——一律以仓库原文为准。
- **不删除任何已有信息**；只做重组、折叠、加引导。原文有的表格/链接/图片都要保留（可移入 `<details>`）。
- **保留中文文件名与相对路径**；改动后逐一点开验证链接和图片可达，不得出现死链。
- 不新增需要外网的图床/CDN；图片继续用仓库内 `硬件资料/接线图/*`。
- `10_review_report.md` 不做学生向重写，只在顶部加一行说明。
- Gitee 兼容：`[!CAUTION]` 等在 Gitee 会降级为普通引用（可接受），`<details>`、任务清单 Gitee 支持。

## 10. Git 提交流程

```bash
git clone https://github.com/carey-hu/engineering-training
cd engineering-training
git checkout -b docs/reading-optimization

# …按本说明逐文件修改…

git add -A
git commit -m "docs: 优化阅读排版与引导逻辑（README + 教程文档 + 示例 README）

- 重排 README 为学习主页：单一起步入口、折叠参考资料
- 教程文档统一结构与上一步/下一步导航
- 安全内容改用 GitHub 警告框 [!CAUTION]
- 不改代码逻辑，不改技术参数"

# 二选一：
git push origin docs/reading-optimization   # 然后在 GitHub 建 Pull Request 审核合并
# 或（确认无误后直接合主干）
# git checkout main && git merge docs/reading-optimization && git push origin main
```

> 建议先走 PR：在 GitHub 的 Files changed 里逐篇预览渲染效果，确认无死链、警告框/折叠块正常，再合并。
